# Правила учебных интенсивов

Документ определяет обязательные правила для материалов репозитория.

## 1. Назначение репозиториев

- `learning` хранит готовые учебные маршруты, задания, лаборатории и конкретные результаты выполнения.
- `knowledge` хранит законченные и обобщённые знания.
- Теоретический материал не дублируется между репозиториями.

## 2. Структура

Иерархия материалов:

```text
раздел → интенсив → спринт → практические артефакты
```

Разделы совпадают с верхнеуровневыми разделами базы знаний.

Каждый раздел может содержать:

- один основной интенсив, покрывающий область целиком;
- несколько дополнительных тематических интенсивов для углубления.

## 3. Основной интенсив

Основной интенсив должен:

- покрывать все основные подразделы области;
- позволять ответить на основные вопросы раздела базы знаний;
- формировать уровень 3 по всей области;
- формировать уровень 4 по ключевым механизмам;
- содержать проверочные спринты;
- завершаться итоговой интеграционной работой.

### Уровень 3

Самостоятельное объяснение механизма, решение типовой задачи, интерпретация результата и применение в знакомой ситуации.

### Уровень 4

Диагностика нетипичной проблемы, сравнение альтернатив, объяснение ограничений, изменение условий эксперимента и связывание нескольких тем.

## 4. Дополнительные интенсивы

Дополнительный интенсив:

- углубляет конкретное направление;
- имеет самостоятельный практический результат;
- указывает необходимые предварительные спринты или интенсивы;
- не повторяет основной интенсив целиком;
- завершается собственной итоговой работой.

Следует использовать предметные названия вроде «Память и производительность», а не абстрактные «Продвинутый уровень».

## 5. Спринт

Спринт является основной единицей прохождения.

Хороший спринт:

- отвечает на один центральный вопрос;
- охватывает связанную группу механизмов;
- обычно занимает 5–10 часов;
- содержит готовый маршрут изучения;
- содержит практические задания;
- содержит одну основную лабораторную работу;
- содержит дополнительный эксперимент или диагностическую задачу;
- заканчивается самостоятельной проверкой;
- приводит к практическому артефакту и полезным заметкам базы знаний.

Если спринт требует более 12–15 часов, его следует разделить.

Спринты не привязываются к календарным неделям.

## 6. Ресурсы

Для каждого спринта заранее подготавливаются четыре группы источников.

### Обязательный маршрут

Минимальный достаточный набор с точными главами, разделами, временными отметками видео и официальной документацией.

### Альтернативные объяснения

Материалы на случай, если основной источник оказался непонятным.

### Углубление

Академические книги, спецификации, статьи, исходный код и дополнительные эксперименты для уровня 4.

### Справочные материалы

RFC, стандарты, man pages, документация языка, ОС и инструментов. Их не требуется читать полностью.

Основной маршрут должен позволять начать работу без дополнительного поиска в интернете.

## 7. Задания

Задания заранее формулируются полностью и содержат:

- цель;
- условия и ограничения;
- ожидаемый результат;
- способ проверки;
- необходимые артефакты.

Задания не должны сводиться к установке инструмента, копированию команд или формальному пересказу текста.

## 8. Лабораторная работа

Лаборатория проверяет понимание механизма, а не знание интерфейса инструмента.

Минимальный цикл:

1. Сформулировать прогноз.
2. Выполнить эксперимент.
3. Собрать наблюдения.
4. Сравнить результат с прогнозом.
5. Изменить одно условие.
6. Объяснить изменение поведения.
7. Сохранить полезный результат.

По возможности указываются окружение, инструменты, проверка результата и очистка созданных ресурсов.

## 9. Проверочные спринты

После каждых 3–5 содержательных спринтов добавляется проверочный спринт.

Он не вводит новую теорию и содержит:

- вопросы по предыдущим темам;
- практическую задачу;
- диагностический сценарий;
- указание, к каким спринтам вернуться при затруднениях.

## 10. Итоговая работа

Итоговый спринт должен:

- объединять несколько предыдущих спринтов;
- давать воспроизводимый результат;
- включать реализацию или комплексный эксперимент;
- включать измерения или наблюдения;
- включать подготовленную неисправность и диагностику;
- требовать объяснения ограничений и компромиссов.

## 11. Связь с базой знаний

В конце каждого спринта указывается:

- какие заметки создать;
- какие заметки дополнить;
- какой вопрос должна раскрывать каждая заметка;
- что находится за её границами;
- с какими заметками её связать.

Отдельный backlog синхронизации не ведётся.

Обобщённое знание хранится в `knowledge`. Код, измерения и результаты конкретного окружения остаются в `learning`.

## 12. Минимум бюрократии

Не являются обязательными:

- статусы и проценты прогресса;
- даты начала и окончания;
- backlog и kanban;
- дневники каждого действия;
- отчёты о времени;
- обязательные ретроспективы;
- отчёты, повторяющие заметки базы знаний.

Текстовый отчёт создаётся только тогда, когда без него нельзя понять или воспроизвести результат.

## 13. Технологии по умолчанию

- Go — основной язык реализаций и экспериментов.
- Linux и Bash — системные наблюдения и автоматизация.
- Docker или Compose — воспроизводимое окружение.
- Git — фиксация результатов.

Допускаются C, ассемблер, Python, SQL, HCL, YAML и другие технологии, когда они необходимы для изучаемого механизма. Go не применяется искусственно.

## 14. Готовность материала

Спринт готов только тогда, когда его можно пройти без самостоятельного проектирования программы и придумывания заданий.

Интенсив готов только тогда, когда его маршрут покрывает заявленную область, содержит проверки и итоговую работу, а обязательные ресурсы позволяют начать прохождение без дополнительного поиска.
