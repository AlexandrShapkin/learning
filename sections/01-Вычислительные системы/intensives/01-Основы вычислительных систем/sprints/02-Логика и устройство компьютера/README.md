# Спринт 02. Логика и устройство компьютера

Ориентировочный объём: 10–14 часов.

## Центральный вопрос

Как из простых логических элементов получаются арифметика, состояние и основные компоненты компьютера?

## Результат спринта

После завершения необходимо уметь:

- строить truth table для комбинационной схемы;
- объяснять NAND, AND, OR, XOR, multiplexer и demultiplexer;
- различать комбинационную и последовательностную логику;
- объяснять half adder, full adder и многоразрядный сумматор;
- связывать ALU control signals с выполняемой операцией;
- объяснять роль clock, register, program counter и memory;
- нарисовать упрощённый datapath «инструкция → регистры → ALU → память»;
- различать архитектуру фон Неймана и Гарвардскую модель на концептуальном уровне.

## Место в интенсиве

Предыдущий спринт объяснил значения битов. Этот спринт показывает, как аппаратные схемы преобразуют и хранят эти биты. Следующий спринт добавит ISA и реальный машинный код.

## Практические задания

### 1. Truth tables

Составить и проверить таблицы для:

- `Xor(a,b)`;
- `Mux(a,b,sel)`;
- `DMux(in,sel)`;
- majority-of-three: результат равен 1, если минимум два входа равны 1.

Для majority-of-three вывести boolean expression и реализовать схему только из NAND.

### 2. Сумматоры

До запуска симулятора вручную рассчитать `sum` и `carry` для всех входов HalfAdder и FullAdder.

После реализации объяснить, почему последовательное соединение full adders создаёт задержку carry propagation.

### 3. ALU control

Для Nand2Tetris ALU создать собственную таблицу:

```text
zx nx zy ny f no -> выражение -> zr -> ng
```

Заполнить минимум 12 полезных комбинаций. Проверить случай `x=-1`, `y=1` и объяснить signed interpretation результата.

### 4. Состояние

Нарисовать timeline минимум на 8 clock cycles для `Register` и `PC`, включая load, increment и reset. Отдельно отметить значение до и после clock edge.

Установить Nand2Tetris Software Suite и создать в `work/nand2tetris/` реализации:

### Ограничения

- Использовать только разрешённые ранее реализованные chips.
- Не копировать реализации из готовых репозиториев.
- Сначала записывать ожидаемый output для одного собственного test case, затем запускать simulator.
- Сохранить только `.hdl` и собственные небольшие `.tst/.cmp`; не коммитить весь software suite.

### Проверка

Все официальные тесты перечисленных chips должны проходить. Дополнительно создать:

1. тест ALU для `x=0x7fff`, `y=1`;
2. тест ALU для `x=0xffff`, `y=1`;
3. тест PC с последовательностью load → inc → reset;
4. тест Register, показывающий, что изменение входа без clock edge не меняет сохранённое состояние.

## Дополнительный эксперимент

Реализовать `RAM8` и измерить не время исполнения simulator, а количество уровней multiplexing/demultiplexing на пути чтения и записи.

Затем нарисовать, как увеличение RAM8 до RAM64 меняет адрес и структуру выбора, но не разрядность слова.

## Самопроверка

1. Почему NAND функционально полон?
2. Где в комбинационной схеме хранится состояние?
3. Почему регистр нельзя заменить обычным проводом с feedback без clocking discipline?
4. Чем ALU отличается от CPU?
5. Что задаёт ISA, а что остаётся решением микроархитектуры?
6. Почему память и процессор не являются просто «быстрой» и «медленной» программой?

Практическая проверка: по заданной таблице control signals ALU определить выражение и flags без запуска simulator.

## Результат для базы знаний

Рекомендуется создать или дополнить:

- **Логические элементы** — boolean functions, gates и композиция; связать с представлением данных.
- **Комбинационная и последовательностная логика** — наличие состояния и роль clock.
- **ALU** — арифметические и логические операции, flags и control signals; связать с инструкциями процессора.
- **Регистры и тактирование** — сохранение состояния, clock edge и program counter.
- **Архитектура компьютера** — CPU, memory, buses и I/O; не углубляться в конкретную материнскую плату.
- **Архитектура фон Неймана** — общая память инструкций и данных, преимущества и ограничения.

## Когда переходить дальше

Можно переходить, когда официальные и собственные тесты проходят, ALU control table объяснима без подсказки, а схема datapath связывает регистры, ALU, память и program counter.
