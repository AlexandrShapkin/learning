# Спринт 30. Retrospectives и improvement experiments

Ориентировочный объём: 18–27 часов.

## Центральный вопрос

Как превращать observations about work into small testable changes rather than recurring discussions and generic action lists?

## Результат спринта

После завершения необходимо уметь:

- create psychologically safe, evidence-based retrospective scope;
- distinguish event, pattern and system condition;
- select one high-leverage problem;
- formulate improvement hypothesis;
- define baseline, intervention and balancing measure;
- limit actions and assign ownership;
- review experiment outcomes and unintended effects;
- retain, adapt or stop practice deliberately;
- avoid blame, ceremony and action-item accumulation;
- use incident and flow data without overfitting.

## Основной маршрут

1. Analyze a synthetic period with delays and defects.
2. Identify patterns and contributing system conditions.
3. Choose one improvement hypothesis.
4. Run simulated before/after history.
5. Decide retain/adapt/stop.

## Лабораторная работа

Создать `work/improvement-experiment/`.

### Requirements

- retrospective begins from observable data and participant perspectives;
- analysis avoids individual blame and single-cause certainty;
- no more than two experiments are active;
- experiment has expected mechanism and balancing metric;
- owner and review date are explicit;
- unchanged context and confounders are documented;
- outcome includes unintended consequences;
- ineffective experiment is stopped rather than ritualized;
- learning updates policy, docs or tooling where appropriate.

### Failure scenarios

1. Same generic action repeats each retrospective.
2. Ten actions have no ownership/capacity.
3. “Communicate better” has no mechanism or measure.
4. Improvement optimizes local speed but worsens quality/queues.
5. Experiment becomes permanent before review.
6. Retrospective turns into individual performance review.
7. One unusual incident drives universal policy.

## Результат для базы знаний

- **Engineering retrospectives and learning reviews**.
- **Process-improvement hypotheses and experiments**.
- **Balancing metrics and unintended consequences**.
- **Retain, adapt or stop decisions**.

## Когда переходить дальше

Можно переходить, когда one process change is evaluated against baseline and balancing evidence and is retained, adapted or removed explicitly.
