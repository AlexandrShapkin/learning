# Спринт 05. Сегментация, jobs и problem hypotheses

Ориентировочный объём: 18–27 часов.

## Центральный вопрос

Какие группы пользователей действительно отличаются по контексту, поведению, ограничениям и требуемому результату?

## Результат спринта

После завершения необходимо уметь:

- сегментировать by behavior, context, need and economics;
- distinguish demographic description from actionable segment;
- formulate jobs, desired outcomes and switching context;
- identify triggers, alternatives and barriers;
- write problem hypotheses by segment;
- estimate segment reach and importance cautiously;
- detect segment overlap and heterogeneity;
- avoid one average user and arbitrary persona groups;
- connect segment to product and go-to-market decisions.

## Основной маршрут

1. Cluster synthetic research evidence by context and behavior.
2. Define 3–5 actionable segments.
3. Write jobs and problem hypotheses.
4. Compare needs, alternatives and adoption barriers.

## Лабораторная работа

Создать `work/segmentation-jobs/`.

### Requirements

- each segment has observable inclusion criteria;
- differences change product, support, pricing or rollout decisions;
- job statement includes context, progress and current alternatives;
- segment size/reach is labeled with evidence quality;
- contradictory behavior inside a segment is reviewed;
- non-user/non-adopter segment is included;
- one initial demographic segment is replaced by behavioral segmentation;
- hypotheses have validation plans.

### Failure scenarios

1. Segment is “users aged 25–40” with no behavior difference.
2. One organization contains multiple jobs but is treated uniformly.
3. Job statement names the desired feature rather than progress.
4. Segment sizes are guessed as precise numbers.
5. Non-adopters and switching costs are absent.
6. Segmentation cannot change any decision.

## Результат для базы знаний

- **Behavioral and needs-based segmentation**.
- **Jobs-to-be-done and desired outcomes**.
- **Switching context, alternatives and barriers**.
- **Segment-specific problem hypotheses**.

## Когда переходить дальше

Можно переходить, когда segments are identifiable, decision-relevant and supported by different observed contexts or behaviors.
