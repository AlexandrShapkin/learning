# Спринт 06. Personas, scenarios и anti-personas

Ориентировочный объём: 17–25 часов.

## Центральный вопрос

Как представить пользовательский контекст для решений, не создавая fictional stereotypes and decorative personas?

## Результат спринта

После завершения необходимо уметь:

- distinguish segment, role, persona and scenario;
- derive personas from evidence;
- include goals, behaviors, constraints and environment;
- write scenarios around tasks and failures;
- use anti-personas/excluded use cases ethically;
- represent accessibility and capability variation;
- avoid demographic decoration and invented quotes;
- maintain provenance and freshness;
- retire persona artifacts that no longer guide decisions.

## Основной маршрут

1. Convert validated segments into minimal decision personas.
2. Write normal, edge and failure scenarios.
3. Add constrained/non-adopter scenario.
4. Audit each persona field for evidence and decision use.

## Лабораторная работа

Создать `work/personas-scenarios/`.

### Requirements

- every persona links to evidence and segment criteria;
- attributes influence a concrete decision;
- scenarios include trigger, context, goal, constraints and outcome;
- accessibility/device/connectivity constraints are represented where relevant;
- anti-persona means unsupported/out-of-scope use, not a stigmatized group;
- fictional biography and decorative demographics are removed;
- update/retirement condition is explicit;
- one scenario exposes a conflict between roles.

### Failure scenarios

1. Persona is invented from stakeholder imagination.
2. Age/name/photo dominate while behavior is vague.
3. Only happy-path scenario exists.
4. Disability or low connectivity is treated as edge afterthought.
5. Anti-persona stereotypes undesirable people.
6. Artifact is never updated after new evidence.

## Результат для базы знаний

- **Segments, roles, personas and scenarios**.
- **Evidence-based personas**.
- **Task, edge and failure scenarios**.
- **Inclusive and constrained user contexts**.

## Когда переходить дальше

Можно переходить, когда each persona/scenario changes a product or rollout decision and every important claim has evidence provenance.
