# Инструкции для ИИ-агентов

Файл определяет порядок разработки материалов репозитория.

## Перед изменениями

1. Прочитать `README.md`, `RULES.md` и `docs/CONCEPT.md`.
2. Изучить актуальный `index.md` соответствующего раздела репозитория `AlexandrShapkin/knowledge`.
3. Проверить существующие интенсивы раздела и не создавать смысловых дублей.
4. Для ресурсов проверить актуальность ссылок и соответствие конкретному спринту.

## Порядок реализации

Интенсивы добавляются по одному. Для каждого интенсива необходимо:

1. Определить цель, границы и предварительные знания.
2. Сопоставить вопросы раздела со спринтами.
3. Сформировать линейный порядок спринтов.
4. Добавить проверочные спринты после каждых 3–5 содержательных.
5. Добавить итоговый спринт.
6. Полностью написать каждый спринт: ресурсы, задания, лабораторию, проверку и результат для базы знаний.
7. Проверить относительные ссылки и структуру каталогов.

## Запрещено

- Создавать пустые интенсивы или спринты-заглушки.
- Оставлять пользователю формулировку основных заданий.
- Давать только список тем без порядка прохождения.
- Добавлять ресурсы без объяснения, для чего они нужны.
- Требовать ведения backlog, таблиц прогресса или дневника.
- Дублировать теоретические статьи из `knowledge`.
- Создавать каталоги `work/` заранее без практических артефактов.

## Качество ресурсов

Приоритет источников:

1. официальная документация и спецификации;
2. признанные учебники и университетские курсы;
3. технические доклады и статьи авторов технологий;
4. качественные альтернативные объяснения.

Для изменчивых технологий необходимо проверять текущие версии и состояние документации перед добавлением.

## Изменения базы знаний

Интенсив может рекомендовать создание и изменение заметок, но не должен автоматически заполнять `knowledge` вместо прохождения, если пользователь отдельно этого не попросил.

Рекомендации должны задавать границы и связи заметки, а не предлагать формальное количество файлов.
